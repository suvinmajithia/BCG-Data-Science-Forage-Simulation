Churn Prediction Model

This notebook outlines the final stage of the customer churn analysis: Predicting customer behavior using the prepared feature set provided (data_for_predictions.csv).

The primary goal is to train a Random Forest Classifier to determine which customers are most likely to churn in the next three months.

Key Objectives:

  1. Model Training: Train the Random Forest model on the finalized feature set.

  2. Performance Evaluation: Use critical metrics—especially Recall and ROC-AUC—to evaluate the model's effectiveness in finding at-risk customers.

  3. Optimization: Adjust the model's prediction threshold to prioritize Recall (finding churners) over strict Precision, providing an actionable tool for the marketing and sales teams.

  4. Final Recommendation: Provide a clear justification for the model's suitability for deployment.

4.2. Model Training and Initial Prediction

  • We will now initialize and train the Random Forest Classifier. Given the class imbalance (many more stable customers than churners), we implement two key strategies:

  1. class_weight='balanced': This critical parameter automatically adjusts the weights in the model to penalize misclassifying the minority class (churners) more heavily. This helps prevent the model from ignoring the churn signal.

  2. n_estimators=100: Using 100 decision trees ensures the model is robust and generalizable.

  • After training (model.fit), we use the test data to generate two key outputs:

  1. y_pred: The hard prediction (0 or 1).

  2. y_pred_proba: The probability scores for each class, which are essential for calculating ROC-AUC and performing threshold tuning later.

In [35]:
# Use n_estimators=100 for a robust model and random_state for reproducibility.

model = RandomForestClassifier(n_estimators=1000, random_state=42, class_weight='balanced')

# Train the model by fitting X_train and y_train
model.fit(X_train, y_train)

# Make predictions on the test set
# 1. Predicted Class (0 or 1)
y_pred = model.predict(X_test)
# 2. Predicted Probability of Churn (1) for ROC-AUC calculation
y_pred_proba = model.predict_proba(X_test)[:, 1]

  • Why did I choose the evaluation metrics that I used?

  • The standard metric, Accuracy, is misleading for a churn problem. Because the dataset is imbalanced (only about 10% of customers churn), a high accuracy score (0.9036) is achieved mainly by correctly predicting the vast majority of stable customers.

  • Therefore, we prioritize:

  1. Recall (Sensitivity): This is the most critical business metric. It measures the percentage of all actual churners that the model correctly identified. High recall minimizes False Negatives (missed opportunities for intervention).

  2. Precision: Measures the percentage of customers the model predicted would churn that actually did. It is important for minimizing False Positives (wasting marketing resources on stable customers).

  3. ROC-AUC (Receiver Operating Characteristic - Area Under Curve): This is the most robust overall metric for imbalanced classification. It measures the model's ability to rank customers by their likelihood to churn, which is vital for targeting campaigns.

  • Justification:

  1. Extremely Low Recall (4.10%): The model is fundamentally failing at the primary task: identifying churners. Out of 366 total actual churners in the test set (351+15), the model only found 15 of them. This leaves 351 False Negatives, meaning 95.9% of at-risk customers would be missed by the intervention program. This is unacceptable for a business trying to save customers.

  2. High Precision Comes at Too High a Cost: The model has an excellent Precision of 0.9375, and only 1 False Positive. This high precision is achieved because the model is extremely cautious about predicting churn. However, for a churn task, we need to balance this: the cost of missing a churner is usually much higher than the cost of reaching out to a few extra stable customers.

  3. Weak Discriminative Power (ROC-AUC of 0.6701): While better than random guessing (0.5), an ROC-AUC below 0.70 suggests the model's ability to reliably rank customers by risk is still weak.

  • Recommendation: The current model is not fit for deployment. We must increase the Recall by sacrificing some Precision through techniques like hyperparameter tuning or adjusting the prediction threshold.

# 1. Define the new threshold
new_threshold = 0.15

# 2. Create new predictions using the lower threshold
y_pred_tuned = (y_pred_proba > new_threshold).astype(int)

# 3. Calculate and print the critical metrics with the new threshold
tn, fp, fn, tp = metrics.confusion_matrix(y_test, y_pred_tuned).ravel()
tuned_recall = recall_score(y_test, y_pred_tuned)
tuned_precision = precision_score(y_test, y_pred_tuned)
tuned_f1 = 2 * (tuned_precision * tuned_recall) / (tuned_precision + tuned_recall)
tuned_cm = confusion_matrix(y_test, y_pred_tuned)

  1. Model understanding
  • A simple way of understanding the results of a model is to look at feature importances. Feature importances indicate the importance of a feature within the predictive model, there are several ways to calculate feature importance, but with the Random Forest classifier, we're able to extract feature importances using the built-in method on the trained model. In the Random Forest case, the feature importance represents the number of times each feature is used for splitting across all trees.
